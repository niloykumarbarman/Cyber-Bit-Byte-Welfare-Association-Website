# Production Fatal Error - Bootstrap Fix

## Error Diagnosis

**Error Message:**
```
Uncaught BadMethodCallException: Method Illuminate\Foundation\Application::configure does not exist
```

**Cause:** 
- Corrupted vendor directory or cached files
- Version mismatch between code and dependencies
- Missing or corrupted bootstrap/cache directory
- Incorrect file permissions

## 🚀 FASTEST FIX (Recommended)

SSH to your production server and run:

```bash
cd /home/cyberbit/welfare.cyberbitbyte.com
bash scripts/production-fix.sh
```

This script will:
1. Set correct permissions
2. Remove vendor and reinstall all dependencies clean
3. Reset code to latest GitHub version
4. Remove ALL cache files
5. Rebuild optimized caches
6. Run migrations
7. Verify server health

The script will automatically verify success at the end. ✓

## Manual Fix Steps (If script doesn't work)

If the script has issues, follow these steps manually:

### Step 1: Connect to Server
```bash
ssh -i your_key.pem cyberbit@your-server-ip
```

### Step 2: Navigate to Project
```bash
cd /home/cyberbit/welfare.cyberbitbyte.com
```

### Step 3: Clean Everything
```bash
# Remove vendor and lock file
rm -rf vendor composer.lock

# Reset git to match GitHub exactly
git fetch origin
git reset --hard origin/main

# Remove all cache files
rm -rf storage/framework/cache/*
rm -rf storage/framework/views/*
rm -rf bootstrap/cache/*

# Fix permissions
chmod -R 777 storage bootstrap/cache
```

### Step 4: Reinstall Clean
```bash
# Fresh install
composer install --no-dev --optimize-autoloader

# Verify
php composer.phar diagnose
```

### Step 5: Clear and Rebuild Caches
```bash
php artisan config:clear
php artisan cache:clear
php artisan view:clear
php artisan route:clear

# Rebuild optimized caches
php artisan config:cache
php artisan route:cache  
php artisan view:cache
```

### Step 6: Run Migrations
```bash
php artisan migrate --force
```

### Step 7: Fix Permissions Again
```bash
chown -R www-data:www-data .
find . -type f -name "*.php" -exec chmod 644 {} \;
find storage -type f -exec chmod 644 {} \;
find bootstrap/cache -type f -exec chmod 644 {} \;
```

### Step 8: Test
```bash
curl https://welfare.cyberbitbyte.com/health
```

You should see:
```json
{"status":"ok","timestamp":"...","app":"Cyber-Bit Welfare"}
```

## If Still Getting Error

Check these in order:

### 1. Check Error Logs
```bash
tail -100 storage/logs/laravel.log
```

### 2. Verify PHP Version
```bash
php --version
# Must be PHP 8.2 or higher
```

### 3. Check Required Extensions
```bash
php -m | grep -E "mbstring|curl|xml|sqlite|pdo"
# All should be present
```

### 4. Verify File Permissions
```bash
ls -la bootstrap/
# bootstrap/cache should be drwxrwxrwx (777)

ls -la storage/
# storage should be drwxrwxrwx (777)
```

### 5. Check Disk Space
```bash
df -h /home/cyberbit
# Ensure space is available
```

### 6. Verify .env File
```bash
cat .env | grep -E "APP_ENV|APP_DEBUG|APP_URL|DB_"
# Should show:
# APP_ENV=production
# APP_DEBUG=false
# APP_URL=https://welfare.cyberbitbyte.com
# DB_CONNECTION=mysql (or sqlite)
```

### 7. Test PHP
```bash
php -r "echo 'PHP Works!';"
# Should print: PHP Works!
```

## Prevention

To prevent this in the future:

1. **Always test before deploying:**
   ```bash
   php artisan serve   # Test locally first
   ```

2. **Use deployment automation:**
   - Push to GitHub
   - Let GitHub Actions test and deploy automatically
   - Automated health checks verify success

3. **Keep composer.lock:**
   - Never delete composer.lock
   - Always commit it to GitHub
   - This ensures exact dependency versions

4. **Monitor error logs:**
   ```bash
   tail -f storage/logs/laravel.log
   ```
